# 盯梢 UX 测试报告 (2026-04-15)

## 测试结果

### ✅ 正常工作
- 落地页正常展示，"开始使用"按钮正常跳转
- 添加/删除监控功能正常
- 变化检测 + AI 摘要工作良好（HN 已积累 10 条高质量摘要）
- 设置 API 正常
- 公网访问正常（~0.86s 响应）

### ⚠️ 需改进
1. **监控列表排序** — 新添加的排在最前面，应该提供排序选项或按创建时间倒序固定
2. **没有空状态引导** — 新用户进来看到空列表，不知道该干什么。需要一个"添加你的第一个监控"的引导
3. **CSS 选择器无提示** — 添加监控时可以填 CSS 选择器，但没有解释这是什么、怎么用
4. **没有手动触发按钮** — 用户添加监控后要等下一次定时执行，没有"立即检查"按钮
5. **删除无确认** — 点删除直接删了（虽然有 confirm，但移动端可能不友好）
6. **移动端适配** — 未专门测试，但 Tailwind 响应式应该基本可用
7. **没有 loading 状态** — 添加监控后页面刷新无 loading 提示
8. **飞书通知未配置提示** — 设置页应该提示"配置飞书 Webhook 开启通知"

### 🔴 V2EX 发帖前必须修复
1. **加"立即检查"按钮** — 用户发帖后试用，等 15 分钟才看到效果会流失
2. **空状态引导** — 新用户打开后台要知道怎么开始
3. **中文示例摘要** — 目前只有英文网页的摘要，需要一个中文的

## 优先修复顺序
1. 立即检查按钮（最影响试用体验）
2. 空状态引导
3. 中文 AI 摘要示例（等 36kr 检测到变化）
