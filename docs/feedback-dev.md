# 盯梢 — 独立开发者视角的真实反馈

> 测试人设：独立开发者，做自己的小产品，日常关注 HN/PH/GitHub Trending，追踪框架更新，偶尔监控论坛讨论。
> 测试时间：2026-04-15
> 测试方式：通过 API 全流程操作（查看监控、查变化历史、添加新监控、手动触发、检验 AI 摘要质量）

---

## 一、实际操作记录

### 1. 查看现有监控列表 ✅
`GET /api/monitors` 返回 8 个监控项，包括 HN、PH、GitHub Trending、36kr、V2EX、竞品页面等。API 响应结构清晰，状态信息完整（active/error + error_message）。

### 2. Hacker News 变化历史 ✅✅
共 10 条变化记录，时间跨度约 4 小时（00:45 → 04:57），平均每 30 分钟检测一次。AI 摘要对每次变化都生成了结构化总结，包含：
- **排名变化**：哪些文章升了降了
- **新增/删除**：哪些文章进了/出了首页
- **数据趋势**：点赞数、评论数增长

**信息密度很高，比我自己刷 HN 效率高很多。** 不用打开页面就能知道过去几小时的动态。

### 3. GitHub Trending 变化历史 ⚠️
`changes: []` — 空的。可能是因为 Trending 页面一天才更新一次，或者监控间隔（60分钟）下内容确实没变。这个倒是合理的，但作为用户我不确定"是没变还是有 bug"。**缺少"最后一次检查内容正常，无变化"的提示。**

### 4. 添加 Next.js Releases 监控 ✅
```json
POST /api/monitors
{ "name": "Next.js Releases", "url": "https://github.com/vercel/next.js/releases", "frequency": 120 }
```
秒级创建，手动触发后成功抓取了完整的 releases 页面内容（包括 canary 版本的 changelog 细节）。

也测试了 CSS selector 功能——用 `.markdown-body` 筛选 Tailwind CSS Releases 页面，成功只提取了 changelog 正文部分（6966字符 vs 全页面通常 2-3 万字符）。**selector 功能实用且好使。**

### 5. 手动触发检查 ✅
`POST /api/monitors/10/check` 响应时间约 6 秒，可以接受。短时间内二次触发没有产生虚假变化，说明 diff 逻辑靠谱。

### 6. AI 摘要质量评估 — **有亮点也有硬伤**

拿 HN 最新一条 change（id=15）做了原始 diff 人工对比：

| 摘要内容 | 实际情况 | 判定 |
|---------|---------|------|
| "guide.world 从第17位消失" | 实际从第17位移到了第19位，**没消失** | ❌ **错误** |
| "Trusted access 新增到第16位" | 原本在第20位，升到了第16位，**是排名上升不是新增** | ❌ **错误** |
| "Ask HN: Easiest UX for Seniors 消失" | 确实不在新内容中了 | ✅ 正确 |
| "OpenAI $852B valuation 消失" | 确实不在新内容中了 | ✅ 正确 |
| "Claude Code 501→515, 评论 294→302" | 精确匹配原始数据 | ✅ 正确 |
| "Dependency cooldowns 从第3升到第2" | 精确正确 | ✅ 正确 |

**摘要准确率大约 70%。** 数字类信息（点赞数、评论数）非常准确；排名变化的大方向对但有混淆"新增"和"排名上升"的问题；最严重的是把"排名下降但仍在榜"误判为"消失"。

Product Hunt 的摘要质量好一些，可能因为结构更规整。但也有几条标记 `AI 摘要暂不可用`（id=2, id=11），说明摘要生成不是 100% 可靠。

## 二、评分与对比

### 痛点解决程度：7/10

**解决得好的：**
- ✅ 不用自己刷页面，自动告诉我"发生了什么变化"
- ✅ HN/PH 这种信息流页面特别适合——我不在意每条内容，只想知道"有啥新的"
- ✅ CSS selector 支持让我可以精准监控 changelog 正文，而不是整个页面的噪音
- ✅ 手动触发 + 定时检查都有，灵活

**没完全解决的：**
- ⚠️ GitHub Trending 检测不到变化（可能频率问题），我作为开发者**最想监控的反而是这个**
- ⚠️ AI 摘要有误判，如果我依赖它做决策（比如发现竞品上了 HN 首页要去看），漏了就亏
- ⚠️ 没有"只通知我关心的关键词"的过滤，比如我只想知道 HN 上有没有出现 "AI agent" 相关的新项目

### 跟现有方案对比

| 维度 | RSS 阅读器 | 手动刷 | Twitter 关注 | 盯梢 |
|------|-----------|--------|-------------|------|
| HN/PH 动态 | 有 RSS 但信息过载 | 有效但浪费时间 | 靠别人转发，延迟大 | **最好** — diff + 摘要 |
| GitHub Trending | 无 RSS | 唯一方式 | 偶尔有人分享 | 理论上好但实测没数据 |
| 框架 Releases | RSS 最佳 | 还行 | 还行 | **接近 RSS**，selector 加分 |
| 论坛讨论监控 | 大多没 RSS | 累 | 不行 | **明显优于其他方案** |
| 竞品价格/功能变化 | 不可能 | 唯一方式 | 不可能 | **杀手级场景** |
| 信息过滤 | 标记已读 | 肉眼 | 算法推荐 | AI 摘要（有时不准） |

**结论：盯梢最大的优势在 RSS 覆盖不到的场景** — 竞品页面监控、论坛讨论、没有 RSS 的网站。对于已有 RSS 的场景（博客更新、Release Notes），RSS 仍然更可靠。

### AI 摘要质量：6/10

**优点：**
- 结构化输出很好（排名变化 / 新增内容 / 删除内容 / 数据更新，分类清晰）
- 数值类信息精确
- 看一眼就能决定要不要深入看原文

**缺点：**
- 会把"排名下降但仍在榜"误判为"删除/消失"——这对监控场景是严重 bug
- 偶尔摘要生成失败，只返回"内容长度变化: X → Y"
- 对 Product Hunt 的投票数变化，很多标注"无变化"但实际显示的数字有差异（可能是 PH 页面的数据展示本身就不一致）
- **没有"这条变化重不重要"的判断** — 所有变化一视同仁，HN 排名微调跟新项目上榜混在一起

### 付费意愿：**愿意，但价格敏感**

| 方案 | 我的判断 |
|------|---------|
| 免费 + 10 个监控 | 会用，够我日常需求了 |
| $5/月，50 个监控 + AI 摘要 | **甜蜜点** — 独立开发者能接受的上限 |
| $10/月 | 除非加关键词过滤 + Slack/飞书推送 + webhook |
| $20/月+ | 不会。这个价位我会自己写脚本 |

**核心考量：** 这是一个"nice to have"而不是"must have"的工具。我之前没有它也活得下去，只是多花点时间刷页面。$5/月 的价格下我会毫不犹豫订阅；$10+ 就要掂量了。

### 最想要但没有的功能

按优先级排序：

1. **🔥 关键词/语义过滤 + 高亮**  
   最核心的缺失。我不想知道 HN 首页每次排名微调，我只想知道"有没有跟 AI agent 相关的新项目上了首页"。让我设置关键词或一段描述，只在匹配时通知我。

2. **📱 推送通知（Slack / 飞书 / Webhook / Email）**  
   现在要自己来 API 查，这跟手动刷页面的本质区别不大。**变化检测 + 不推送 = 功能做了一半。**

3. **📊 变化时间线可视化**  
   能看到一个网站过去一周的变化频率、变化幅度的趋势图。帮我判断"这个页面值不值得继续监控"。

4. **🔗 跟 RSS 互补而不是替代**  
   让我同时配置 RSS 源和网页监控，统一在一个地方看。

5. **🤖 更聪明的 diff**  
   现在是纯文本 diff，对于动态内容（时间戳、计数器）产生大量噪音。应该有"忽略数字变化"或"只关注新增内容"的选项。

6. **📦 批量导入/导出**  
   让我一次性导入 20 个 URL，或者导出配置给别人用。

### 会推荐吗？

**V2EX：会发一个"分享"帖。** 适合推荐的点：
- 独立开发者监控竞品 — 这个角度在 V2EX 会引起共鸣
- 不用写代码就能监控网页变化 — 对非技术产品经理也有价值
- AI 摘要这个卖点 — 虽然不完美但方向对

**推特：暂时不会。** 推特上这类工具太多了（Visualping、ChangeTower、Distill.io），需要有一个明确的差异点。如果加上关键词过滤 + 中文场景优化，我会在推特发。

## 三、总结

盯梢解决了一个真实但不紧急的痛点。对独立开发者来说，最有价值的场景是**竞品监控**和**无 RSS 网站的追踪**，而不是替代 RSS 阅读器。

**最大的亮点：** CSS selector + AI 摘要的组合，让网页监控从"告诉你变了"升级到"告诉你变了什么"。

**最大的软肋：** AI 摘要准确率不够高（~70%），且缺少推送和关键词过滤——这意味着我还是要主动来查，而且看到的信息可能不完全准确。

**一句话：方向对，做到了 60 分的可用。加上推送和关键词过滤能到 80 分。AI 摘要准确率提上去能到 90 分。**
