# 盯梢 — 三角色试用反馈汇总

## 综合评分

| 角色 | 评分 | 付费意愿 | 核心结论 |
|------|------|---------|---------|
| 🧑‍💼 产品经理 | 6/10 | ¥29-99/月（需解决SPA） | AI摘要是杀手功能，但SPA抓不了+没推送是致命伤 |
| 👨‍💻 独立开发者 | 7/10 | $5/月 | 方向对，HN/PH体验好，但AI准确率~70%，缺关键词过滤 |
| 🛒 电商运营 | 2/10 | ¥30-200/月（需解决抓取） | 电商场景基本不可用，8个监控只有1个政府网站成功 |

## 共识问题（3个角色都提到的）

### 🔴 P0 — 必须解决
1. **JS渲染页面抓不到** — 当前用 fetch+cheerio 纯 HTTP 请求，国内主流网站（淘宝/京东/飞书/小米）全部失败。需要集成无头浏览器（Playwright/Puppeteer）
2. **没有推送通知** — 变化检测了但不通知用户，等于功能做了一半。飞书webhook代码写了但没人配置、没引导

### 🟡 P1 — 强烈期望
3. **AI摘要准确率不够** — 会把"排名下降仍在榜"误判为"消失"，约70%准确率
4. **关键词/语义过滤** — 用户不想看所有变化，只想看匹配关键词的变化

### 🟢 P2 — 锦上添花
5. 变化重要性分级（不是所有变化都值得通知）
6. 更多通知渠道（微信/邮箱/Slack）
7. 批量导入URL
8. 变化趋势可视化

## 关键洞察

1. **AI摘要是差异化优势** — 所有角色都认可"告诉你变了什么"比"告诉你变了"有价值得多
2. **静态HTML网站体验很好** — HN、ProductHunt、GitHub、政府网站等效果优秀
3. **JS渲染是最大的墙** — 国内80%+的商业网站是SPA，不解决这个等于放弃大部分用户
4. **推送是把"能用"变成"好用"的关键** — 没推送用户还是要主动来查
5. **定价甜蜜点** — 个人用户 ¥15-30/月，团队用户 ¥99-200/月

## 行动优先级

| 优先级 | 任务 | 预估工作量 | 影响 |
|--------|------|-----------|------|
| P0 | 集成 Playwright 解决 JS 渲染 | 1-2天 | 覆盖从20%→80%的网站 |
| P0 | 完善飞书推送 + 新用户引导配置 | 半天 | 把"能检测"变成"能通知" |
| P1 | 优化 AI prompt 提升摘要准确率 | 半天 | 从70%→90%准确率 |
| P1 | 关键词过滤功能 | 1天 | 减少噪音，提升信噪比 |
| P2 | 更多通知渠道 | 1-2天 | 扩大用户群 |
